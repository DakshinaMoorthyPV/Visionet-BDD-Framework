<img src="https://badgen.net/badge/Setup/Guide/006daf?icon=book" alt="Setup Guide Badge"> &nbsp;
    <h1>Framework Setup and Version Control Process</h1>
     <h2>This Guide Covers Everything from Cloning the Repository from Git to Configuring Your Environment</h2>
   	 <h3>Steps to Set Up the Framework</h3>
   	 <h4>1. Install Git</h4>
    <p>Download and install Git from <a href="https://git-scm.com/downloads" style="color: #0000EE; text-decoration: underline;">here</a>.</p>
    <h4>2. Clone the Repository</h4>
    <p>Open your terminal or command prompt and run:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git clone https://github.com/DakshinaMoorthyPV/Visionet-BDD-Framework.git</pre>
    <h4>3. Navigate to the Project Directory</h4>
    <p>Change to the project directory:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">cd Visionet-BDD-Framework</pre>
    <h4>4. Import the Cloned Maven Project</h4>
    <p>In Eclipse, follow these steps:</p>
    <ul>
        <li>Go to <strong>File</strong> > <strong>Import...</strong></li>
        <li>Select <strong>Maven</strong> > <strong>Existing Maven Projects</strong> and click <strong>Next</strong></li>
        <li>Browse to the directory where you cloned the project (e.g., <em>Visionet-BDD-Framework</em>)</li>
        <li>Ensure the <em>pom.xml</em> file is selected and click <strong>Finish</strong></li>
    </ul>
    <h4>5. Create a New Branch from Master</h4>
    <p>Create and switch to a new branch:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git checkout -b your-branch-name</pre>
    <h4>6. Push the Branch to Remote</h4>
    <p>Push your new branch to the remote repository:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git push -u origin your-branch-name</pre>
    <h4>7. Make Changes and Commit</h4>
    <p>Stage and commit your changes:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git add .<br>git commit -m "Your commit message"</pre>
    <h4>8. Push Changes to Your Branch</h4>
    <p>Push your committed changes to the remote repository:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git push</pre>
    <h4>9. Merge Your Branch into Master</h4>
    <p>Switch to the master branch:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git checkout master</pre>
    <p>Pull the latest changes from the remote master branch:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git pull origin master</pre>
    <p>Merge your branch into master:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git merge your-branch-name</pre>
    <h4>10. Handle Merge Conflicts</h4>
    <p>If there are merge conflicts, edit the conflicted files to resolve the conflicts. After resolving, stage the changes:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git add .</pre>
    <p>Continue the merge process:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git commit</pre>
    <h4>11. Push Merged Changes to Remote Master</h4>
    <p>Push the updated master branch to the remote repository:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git push origin master</pre>
    <h4>12. Pull Latest Changes to Your Branch</h4>
    <p>If you need to pull the latest changes from the remote master branch into your branch:</p>
    <pre style="background-color: #f4f4f4; padding: 10px;">git checkout your-branch-name<br>git pull origin master</pre>
