# Visionet Automation Suite Documentation

## Folder Structure Overview

This guide outlines the folder and file structure of the Visionet Automation Suite, designed to facilitate understanding and navigation for developers and testers.

### `src/main/java`
- **`com.visionetsystems.framework.pages`**: Contains classes for web and mobile page interactions, categorized by specific platforms (`appium`, `web`) and uses (`generic`, `interface`).
- **`com.visionetsystems.framework.pagefactory`**: Factory classes for initializing page objects, aiding in modular architecture by separating page construction from test logic.
- **`com.visionetsystems.framework.selenium`**: Encapsulates Selenium-specific classes, enhancing Selenium's capabilities with custom actions and utilities.
- **`com.visionetsystems.framework.utils`**: Utility classes providing essential services like configuration reading, report generation, and logging.

### `src/main/resources`
- **`apps`**: Mobile applications, e.g., `.apk` files for Android.
- **`drivers`**: Executables for browsers such as Chrome, Firefox, and Edge.
- **`fonts`**: Custom fonts for reports or UI testing.
- **`Report_Images`**: Images used in automated reports or documentation.
- Configuration files (`browserstack.yml`, `extent-config.xml`, `log4j.xml`) manage reporting and logging.

### `src/test/java`
- **`com.visionetsystems.framework`**:
  - **`listeners`** and **`runner`**: TestNG listeners and test runners.
  - **`stepdefinitions`**: BDD step definitions linking Gherkin syntax to executable Java code.

### `src/test/resources`
- **`features`**: BDD feature files defining application behavior in a readable format.
- **`features/config`**: Configuration files for specific testing sprints or features.

### Maven Dependencies and JRE System Library
- Java libraries and Maven dependencies required for compiling and running the suite.

### Sprint-Execution-Reports
- **`extent-reports`**, **`logs`**, **`screenshots`**, **`summaryreports`**: Outputs from test execution including detailed logs and summary reports.

### Test Data Management

<img src="https://badgen.net/badge/app.config.json/Update%20It/red?icon=script" alt="Installation Badge">


The `test-automation-config.xml` file contains the following configuration settings for test data management:

- **URL**: The base URL of the application under test.
- **Browser**: The browser to be used for test execution.
- **Mail Settings**: Configuration for sending test reports via email.

Here is an example of the `app.config.json` file structure:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd">
<properties>
    <!-- General Test Data Management Configuration -->
    <entry key="URL">https://example.com</entry>
    <entry key="Browser">chrome</entry>
    
    <!-- Mail Settings Configuration for Sending Test Reports -->
    <entry key="MailSettings.SendReport">true</entry>
    <entry key="MailSettings.Recipients">user1@example.com, user2@example.com</entry>
    <entry key="MailSettings.Subject">Test Execution Report</entry>
    <entry key="MailSettings.SMTPServer">smtp.example.com</entry>
    <entry key="MailSettings.SMTPPort">587</entry>
    <entry key="MailSettings.SMTPUsername">test@example.com</entry>
    <entry key="MailSettings.SMTPPassword">password123</entry>
</properties>

```
You can customize the values according to your specific requirements.

## Getting Started with Selenium, Appium, and BDD using TestNG and Cucumber

### 1. Installing Dependencies
Before running tests, ensure all necessary dependencies are installed. For Java projects using Maven, dependencies like Selenium WebDriver, Appium, TestNG, and Cucumber should be included in your `pom.xml`.

```xml
<!-- Example dependencies for Selenium and Appium with Cucumber and TestNG -->
<dependencies>
    <!-- Selenium WebDriver -->
    <dependency>
        <groupId>org.seleniumhq.selenium</groupId>
        <artifactId>selenium-java</artifactId>
        <version>LATEST_VERSION</version>
    </dependency>
    <!-- Appium Java Client -->
    <dependency>
        <groupId>io.appium</groupId>
        <artifactId>java-client</artifactId>
        <version>LATEST_VERSION</version>
    </dependency>
    <!-- TestNG -->
    <dependency>
        <groupId>org.testng</groupId>
        <artifactId>testng</artifactId>
        <version>LATEST_VERSION</version>
        <scope>test</scope>
    </dependency>
    <!-- Cucumber for TestNG -->
    <dependency>
        <groupId>io.cucumber</groupId>
        <artifactId>cucumber-testng</artifactId>
        <version>LATEST_VERSION</version>
        <scope>test</scope>
    </dependency>
</dependencies>
```

3. Locate the `test-Reports/html-report/[Date]/Test_Run_Id [Number]` directory.

4. Within the `Test_Run_Id [Number]` directory, you will find an HTML file named `mochawesome.html`.

5. Double-click on the `mochawesome.html` file to open it in your default web browser.

6. The Mochawesome test report will now be displayed in your browser. It will provide a comprehensive overview of your test results, including test suites, test cases, assertions, and any associated screenshots or videos.
